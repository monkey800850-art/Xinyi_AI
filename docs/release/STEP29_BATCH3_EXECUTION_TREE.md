# STEP29 第三批执行任务树（只规划，不执行）

## 1. 第三批目标与范围

### 1.1 本批目标
- 建立“缺陷卡驱动”的稳定化节奏，支持后续问题快速开工与回归闭环。
- 补齐第二批遗留 P1 平台能力：会计分录批量导入、账套导入。
- 推进 Payroll/Tax 从“最小计算 API”走向“可落流程 API（先 API 后 UI）”。
- 延续第二批回归与 UI 全链测试方法，防止后续断篇或跳步。

### 1.2 本批不做
- 不做 AI/OCR/语音识别相关能力。
- 不重构凭证主流程、不重构全量薪资模块。
- 不做跨批次大规模历史数据治理。
- 不在本规划中直接执行代码实现。

## 2. 第三批任务树（执行顺序）

执行主顺序：`3A -> 3B -> 3C -> 3D`

- 前置依赖：3A-T2。

## 3B P1 平台能力补齐

### 3B-T1 会计分录批量导入（API优先）
- 交付物：
  - 导入模板定义文档
  - 预检 API + 导入 API（设计与实现分步）
  - 对应测试清单
- 最小验收：
  - 支持至少 1 种标准模板导入。
  - 预检能返回行级错误（科目、借贷平衡、期间）。
  - 导入结果可审计、可追溯批次。
- 风险边界：
  - 不跳过校验直接落账。
- 前置依赖：3A-T2（缺陷与回归节奏已建立）。

### 3B-T2 账套导入（业务级）
- 交付物：
  - 账套导入规范（快照结构/版本）
  - 导入校验 API（与现有 `backup-restore-verify` 风格兼容）
- 最小验收：
  - 导入前校验可拦截结构错误、重复编码冲突。
  - 成功导入后可做初始化完整性复核。
- 风险边界：
  - 不覆盖线上账套，默认“新账套导入”策略。
- 前置依赖：3B-T1（复用批处理/校验框架）。

### 3B-T3 导入审计与回滚策略（最小版）
- 交付物：
  - 导入批次审计字段定义
  - 导入失败回滚策略说明
- 最小验收：
  - 导入行为可在审计中定位操作人、时间、批次、结果。
  - 失败批次不产生半成品数据。
- 风险边界：
  - 不实现复杂多版本回滚，先保证事务一致性。
- 前置依赖：3B-T1、3B-T2。

## 3C Payroll/Tax 进阶实现（先 API 后 UI）

### 3C-T1 工资期间与工资单最小流程 API
- 交付物：
  - 期间管理 API（开关/锁定）
  - 工资单草稿/确认 API
- 最小验收：
  - 可完成“草稿 -> 计算 -> 确认”闭环。
  - 期间锁定后禁止变更。
- 风险边界：
  - 不做复杂考勤联动。
- 前置依赖：3A-T2。

### 3C-T2 税务台账衔接
- 交付物：
  - 税务台账结构与写入服务
  - 期间/员工维度查询 API
- 最小验收：
  - 能复核每期税额来源（字段快照 + 计算版本）。
- 风险边界：
  - 不覆盖申报系统对接，仅内部台账。
- 前置依赖：3C-T1。

### 3C-T3 凭证建议映射
- 交付物：
  - Payroll->凭证建议映射规则文档
  - `preview/suggest` 调用链联调说明
- 最小验收：
  - 输出建议草稿，不自动入账。
  - 复用 STEP28-B-T1 规则校验。
- 风险边界：
  - 不改凭证审批/过账主流程。
- 前置依赖：3C-T1、3C-T2。

### 3C-T4 年终奖 merge 口径增强
- 交付物：
  - merge 口径升级说明（由示例口径扩展到累计口径）
- 最小验收：
  - 能输入累计参数并输出可解释计算过程。
- 风险边界：
  - 不一次性覆盖所有政策分支。
- 前置依赖：3C-T2。

### 3C-T5 劳务报酬规则扩展
- 交付物：
  - 扩展规则文档（含汇缴差异预留位）
- 最小验收：
  - 保持现有 API 兼容，新增参数可选。
- 风险边界：
  - 不引入非居民复杂规则。
- 前置依赖：3C-T2。
