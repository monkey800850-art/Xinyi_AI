# STEP29-3A 缺陷修复卡（模板 + 执行流程）

## 1. 使用场景与适用范围
- 适用阶段：第三批及后续批次。
- 适用对象：功能缺陷、校验缺陷、权限缺陷、回归缺陷。
- 适用分级：`P0 / P1 / P2`。
- 使用原则：缺陷先登记卡，再执行修复；无卡不修复、无回归不关卡。

## 2. 缺陷卡模板字段（执行卡）

```md

- 复现步骤：

- 实际结果：
- 预期结果：

- 影响范围：账套/用户角色/数据范围/报表范围
- 根因初判：

- 负责人：
- 计划完成时间：
- 状态：Open / In Progress / Resolved / Verified / Closed / Reopen
```

## 3. 修复执行流程（发现 -> 分级 -> 修复 -> 复测 -> 关闭）
1. 发现：记录缺陷卡，附证据路径。
2. 分级：按分级规则判定 P0/P1/P2。
3. 修复：按“最小变更”原则处理，不扩散重构。
4. 复测：先跑缺陷最小回归，再跑关联回归矩阵项。
5. 关闭：满足验收标准且文档留痕后关闭；复测失败则 Reopen。

## 4. 缺陷分级规则（沿用当前标准）
- `P0`：阻断主流程、财务结果错误、权限绕过、500/崩溃、数据破坏风险。
- `P1`：功能可绕过但明显影响使用，或校验缺失导致高概率误操作。
- `P2`：体验、文案、提示一致性、轻微流程不顺。

## 5. 立即修复触发条件
- 任一 `P0` 缺陷确认后立即进入修复通道：
  - 当日建卡并指派负责人；
  - 冻结非必要需求并行；
  - 修复后必须完成“最小回归 + 关键链路回归”再解冻。

## 6. 示例（样例卡）

### [DEFECT-20260303-001] 模板建议未拦截关闭期间（样例）
- 缺陷编号：DEFECT-20260303-001
- 模块：凭证模板建议
- 发现阶段：UAT
- 复现步骤：
  1. 关闭 `2025-02` 会计期间；
  2. 调用模板建议接口，`biz_date=2025-02-15`；
  3. 观察返回结果。
- 实际结果：返回 `200` 并生成建议。
- 预期结果：返回 `400`，提示期间已结账。
- 严重级别：P1
- 证据路径：`/tmp/uat_case_defect_001.json`
- 影响范围：凭证录入前校验链，影响账期控制一致性。
- 根因初判：模板建议链路未复用期间校验函数。
- 修复策略（最小变更）：复用既有期间校验逻辑，不改主流程。
- 回归范围：模板建议 4 组 + 关键报表 1 组。
- 验收标准：
  - 关闭期间调用返回 `400`；
  - 开放期间调用保持 `200`；
  - 回归集全通过。
- 状态：Open
